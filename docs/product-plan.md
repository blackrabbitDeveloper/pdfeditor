# PDF Editor 제품 기획

## 제품 한 줄 정의

서버 업로드 없이 브라우저에서 PDF 페이지를 빠르게 재구성하고 다시 저장하는 GitHub Pages 웹 유틸리티.

## 해결하려는 문제

- PDF 페이지 순서를 바꾸거나 일부 페이지만 제거하려고 무거운 데스크톱 앱을 설치해야 한다.
- 민감한 문서를 알 수 없는 온라인 서버에 업로드하기 어렵다.
- 여러 PDF를 합치는 단순한 작업에도 복잡한 편집기가 필요하다.

## 핵심 원칙

1. 로컬 우선: 파일은 브라우저 메모리에서만 처리한다.
2. 즉시 이해: 썸네일을 직접 옮기는 방식으로 조작한다.
3. 설치 없음: 최신 데스크톱/모바일 브라우저와 GitHub Pages에서 동작한다.
4. 원본 보존: 결과는 새 파일로 내려받고 원본 파일은 변경하지 않는다.

## 1차 MVP

- PDF 클릭 선택 및 드래그 앤 드롭 열기
- 여러 PDF 이어 붙이기
- 페이지 썸네일과 다중 선택
- 드래그로 페이지 순서 변경
- 선택 페이지 왼쪽/오른쪽 회전
- 선택 페이지 복제 및 삭제
- 편집 결과를 새 PDF로 저장
- 파일명, 페이지 수, 처리 상태 표시
- 키보드 Delete, Ctrl/Cmd+A 지원

## 후속 단계

- 2차: 1페이지 상세 보기, 페이지 탐색과 확대/축소, 텍스트, 이미지, 서명, 자유 그리기 오버레이
- 3차: 페이지 추출, 분할 저장, 워터마크, 페이지 번호
- 4차: 실행 취소/다시 실행, 암호화 PDF 안내, 대용량 최적화

## 기술 구성

- Vanilla HTML/CSS/JavaScript: 별도 빌드 없이 GitHub Pages 배포
- PDF.js: 페이지 미리보기 렌더링
- pdf-lib: 페이지 복사, 회전, 병합 및 결과 저장
- Node 내장 테스트 러너: 순수 페이지 배열 로직 검증

## 완료 기준

- 두 개 이상의 PDF를 불러와 섞은 뒤 하나의 정상 PDF로 저장할 수 있다.
- 회전, 복제, 삭제, 재정렬 결과가 저장 파일에 그대로 반영된다.
- 빈 문서 저장과 마지막 페이지 삭제를 방지한다.
- 모바일 화면에서도 파일 열기와 기본 작업이 가능하다.
